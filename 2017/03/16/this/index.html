this

2017-03-16
  • javascript
  • udemy

이번 시간에는 this 에 대해 알아보자.

보통 this 를 떠올리면,
가끔 this 가 어디서 왔는지, 어디로 가는지 혼동될 때가 너무나 많다.

this 가 어디서 왔는지 알아가다 보면 Execution Context 라는 관문이 나온다.
그 관문을 통과하면, 다른 관문들을 줄지어 만나게 된다.

Variable Object => Scope Chain => Variable Instantiation => this value => Execution ..

차근차근 하나씩 짚어보자. 이제 소개할 내용은 자바스크립트의 동작원리 의 일부를 요약한 것이다.

Execution Context (EC)

EC는 추상적으로 예기하면, 코드가 실행(Execution)되는 영역(Context)이다.
EC는 Global Execution Context (GEC) 와 Function Execution Context (FEC) 로 나누어진다.
GEC 는 쉽게 말해 브라우저 탭을 열 때 마다 하나씩만 생성되며,
FEC 는 함수가 실행될 때 생성된다. GEC 는 여러 FEC 를 둘 수 있다.

EC 는 결국 object 라고할 수 있다. GEC 가 부모 객체이며, FEC 는 GEC의 자식 객체인 것이다.
근데 각 GEC/FEC 마다 숨겨놓은 자식들이 더 있다.. 설마 바람피운건 아니겠지

그 자식들은 바로 아래 3명이다.

  • Variable Object
  • Scope Chain
  • this value

Variable Object (VO)
VO 는 변수, 매개변수, 인수, 함수 선언을 가진다. (함수 Expression 은 제외)

Scope Chain (SC)
SC 는 상위 컨텍스트(GEC/FEC)에 대한 변수에 접근하기 위한 목적으로 상위 VO 에 대한 참조를 체인 형태로 가지고 있다.

여기서 잠깐!
W3Schools - Scope

In JavaScript, scope is the set of variables, objects, and functions you have access to.

자바스크립트에서, 스코프란 접근 가능한 변수, 객체 그리고 함수의 집합이라 할 수 있다.

특정 함수 내에서 참조할 수 있는 함수 내/외부상의 접근 가능한 변수, 객체, 함수를 의미한다는 정도로 이해하자. 더 어렵잖아;

this value
(오늘의 주인공) this value 는 GEC/FEC 를 포함하고 있는 객체를 참조하는 변수이다.

EC, GEC, FEC, VO, SC .. 등은 this 의 쓰임새를 알기 위해 무조건 알아야 하는 것은 아니므로,
가볍게 읽고 넘어가도록 하자.

코드로 좀 더 구체적으로 배워보자.
JavaScript Garden에서는 this 가 가리킬 수 있는 객체는 정확히 5가지라고 정의하고 있다.
하지만 몇 가지가 더 있긴한 것 같다.

1.1. Global Scope 에서 호출 시

1.2. 함수 호출 시

**1.3. IIFE (즉시 실행 함수)일 때

2. 생성자 호출시

3. 객체의 메소드 호출 시

4. this 를 정해주는 경우

5.1. 함수가 이벤트 핸들러로 사용될 때, this 는 이벤트가 발생한 엘리먼트로 설정된다.

5.2. in-line 이벤트 핸들러에서 this 에 값이 할당되었을 때

6. 상속(부모의 속성을 포함한다)

7. Global 함수에 객체 bind

this 의 함정

1. this 는 Global 객체를 가리킨다. (Javascript 의 잘못 설계된 부분이라는 의견도 있다.)

1번에 대한 해결책(1): Local 변수(self)의 사용

1번에 대한 해결책(2): bind 메서드의 사용 (ECMAScript5 이상)

2. 메서드를 다른 변수에 할당

2번에 대한 해결책

3. 객체 속성에서의 this 사용

4. in-line 이벤트 핸들러에서 this 에 값이 할당되지 않았을 때(return this;)

**5. strict 모드일 경우

5번에 대한 해결책

마치며
this 가 가리키는 5가지 객체와, 4가지 함정을 예제들을 통해 간단히 살펴보았다.
실무에서는 이외에도 다양한 케이스가 있을거라고 생각한다.
필자는 this의 원리를 정확히 이해못하고 대부분 로컬변수 self 에 this 를 대입해서 사용한 경우가 많았다.
