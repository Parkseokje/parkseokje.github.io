이번 시간에는 this 에 대해 알아보자.

보통 this 를 떠올리면,
가끔 this 가 어디서 왔는지, 어디로 가는지 혼동될 때가 너무나 많다.

this 가 어디서 왔는지 알아가다 보면 Execution Context 라는 관문이 나온다.
그 관문을 통과하면, 다른 관문들을 줄지어 만나게 된다.

Variable Object => Scope Chain => Variable Instantiation => this value => Execution ..

차근차근 하나씩 짚어보자. 이제 소개할 내용은 자바스크립트의 동작원리 를 요약 정리한 것이다.

Execution Context (EC)

EC는 추상적으로 예기하면, 코드가 실행(Execution)되는 영역(Context)이다.
EC는 Global Execution Context (GEC) 와 Function Execution Context (FEC) 로 나누어진다.
GEC 는 쉽게 말해 브라우저 탭을 열 때 마다 하나씩만 생성되며,
FEC 는 함수가 실행될 때 생성된다. GEC 는 여러 FEC 를 둘 수 있다.

EC 는 결국 object 라고할 수 있다. GEC 가 부모 객체이며, FEC 는 GEC의 자식 객체인 것이다.
근데 각 GEC/FEC 마다 숨겨놓은 자식들이 더 있다.. 설마 바람피운건 아니겠지

그 자식들은 바로 아래 3명이다.

  • Variable Object
  • Scope Chain
  • this value

Variable Object (VO)
VO 는 변수, 매개변수, 인수, 함수 선언을 가진다. (함수 Expression 은 제외)

Scope Chain (SC)
SC 는 상위 컨텍스트(GEC/FEC)에 대한 변수에 접근하기 위한 목적으로 VO 에 대한 참조를 차례대로 정돈하여 가지고 있다.

this value
(오늘의 주인공) this value 는 GEC/FEC 를 소유하고 있는 객체를 참조하는 변수이다.
window (global object) 객체는 GEC를 포함하므로 GEC의 this value는 window 를 참조하게 된다.
