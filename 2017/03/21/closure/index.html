이번 시간에는 Closure (클로저) 에 대해 알아보자.
먼저 MDN - Closure 의 정의를 보면

클로저는 독립적인 (자유) 변수 (지역적으로 사용되지만, 둘러싼 범위 안에서 정의된 변수)를 참조하는 함수들이다. 다른 말로 하면, 이 함수들은 그들이 생성된 환경을 ‘기억’한다.

음.. 조금 많이 어렵다. 차근차근 정의를 살펴보자.

  1. 클로저는 독립적인 (자유) 변수를 참조하는 함수이다.
  2. 다른 말로, 클로저는 그들이 생성된 환경을 기억한다.

자유변수를 기억한다는 의미인가 보군. 라고 생각했다면 당신은 천재

예를 들어,

위 코드를 실행하면 1 이 나온다. 함수가 종료되고 나면 함수안의 변수 a도 메모리에서 사라진다.

이렇게 하면 어떻게 될까? test 함수는 종료되었으니, a 도 사라졌을 것이다.
근데 해보면 1 이 찍힌다. 대충 그럴거 같더라ㅎㅎ

이때 a 가 바로 자유변수이다. 그리고 test(); 를 통해 반환된 함수 표현식(function expression) return function() .. 이 바로 클로저 이다.
이 함수가 자유변수인 a 를 기억한다는 의미인 것이다. 쉽게 말하면 자유변수를 훔쳐서 나온거지

다시 한번 MDN - Closure 정의를 살펴보자.

클로저는 독립적인 (자유) 변수 (지역적으로 사용되지만, 둘러싼 범위 안에서 정의된 변수)를 참조하는 함수들이다. 다른 말로 하면, 이 함수들은 그들이 생성된 환경을 ‘기억’한다.

조금 감이 오는가? 그렇다면 심화예제로 넘어가보자.

우선 하나씩 살펴보자.
arr 는 배열이다. 배열안에는 무엇이든 넣을 수 있는데, 위의 경우 i값을 출력하는 함수 표현식(function expression) 을 3번 넣고(arr.push) 있다. (넣기만 할 뿐 함수가 실행되진 않는다.)
그리고 arr 을 return 하고 있다.

그렇다면 여기서 클로저가 있을까? 있다면, 과연 어디에 있을까?

클로저는 자유 변수를 기억하는 함수라고 했다.

즉, 최초 함수(test)의 실행이 끝난 뒤 반환되는 함수(들)일 것이다. 그렇다면, arr 변수에 들어있는 함수 표현식이 바로 클로저임을 알 수 있다. arr 은 위장이야 위장
근데 이상한 점이 있다. 왜 모두 3을 출력할까?

쉽게 설명하면,
클로저가 가지고 나온 자유변수 i 는 일종의 메모리 주소, 즉 참조라고 할 수 있다.
클로저 즉 함수를 실행할 때 비로서 자유변수 메모리에 있는 값을 꺼내게 되므로, 이때의 i 의 값은 for loop 이 완전히 끝난 상태의 결과값인 3이 되는 것이다.

만약 클로저 마다 실행당시의 자유변수의 값을 유지하고 싶다면 어떻게 해야 할까?

IIFE 는 즉시 실행되는 함수 표현식이다. 이전에 자유변수였던 i 를 j 를 통해 주입받고 있다.
그리고 마지막에 또 다른 함수표현식을 리턴하고 있다. 이 놈이 또 숨어있네
클로저가 보이는가? 그렇다. 바로 IIFE 안에 있는 return 되고 있는 함수 표현식이 바로 클로저 가 되겠다.

이전과 다른 점은, 이번에는 i 가 아니라 j 를 참고하게 되므로, 정상적인 값이 출력되는 것을 알 수 있다.

이번에는 setTimeout 을 이용한 예제를 살펴보자.

sayHiThreeSecondsLater 함수는 실제로 아주 금방 끝나버리지만
setTimeout 은 this function has finished 가 출력되고, 3초 뒤에 callback (콜백) 함수 (이놈이 클로저) 를 실행하는 것을 알 수 있다.
콜백함수에 대한 자세한 내용은 Yuby’s Lab 를 참고하자.

다시 정의를 살펴보자

클로저는 독립적인 (자유) 변수 (지역적으로 사용되지만, 둘러싼 범위 안에서 정의된 변수)를 참조하는 함수들이다. 다른 말로 하면, 이 함수들은 그들이 생성된 환경을 ‘기억’한다.

환경이란 바로 클로저 주의의 실행환경을 의미한다.
위 예제에서 실행환경(Execution Context) 은 모두 2 개이다.

그럼 이번엔 클로저를 이용한 Function Factory (함수 팩토리) 예제를 알아보자.
팩토리 함수 (객체를 반환하는 함수)라고도 있는데, 함수 팩토리와의 차이점은 객체를 반환하느냐 함수 표현식(클로저)를 반환하느냐의 차이가 있다.
팩토리 함수에 대한 자세한 내용은 codepink 을 참고하자.

다음 예제는 makeGreeting 함수의 인자(argument)인 언어(language)에 따라 다른 인사말을 다르게 출력하는 예제이다.

greet.. 변수들은 각각의 실행환경(makeGreeting)을 생성하고, 각각의 실행환경이 소멸될 때에 남겨진 클로저와 자유변수들을 가진다.
각 자유변수는 실행환경 마다 고유값(language)을 가지고 있는 상태이다.

greet.. 변수들은 각자의 클로저를 가지게 되고,
클로저인 function(firstname, lastname).. 함수 표현식은 자신이 참조하고 있는 자유변수에 따라 다른 값을 출력하게 된다. 참 쉽죠잉?

makeGreeting 은 함수 오버로딩(overloading, wikipedia - 함수 오버로드) 을 구현하고 있다고 보면 된다. (사실 JavaScript 는 함수 오버로딩을 지원하지 않는다)
물론 인자(language)가 모두 동일하고, 인자의 개수가 다를 경우 체크하는 로직이 없지만 같은 함수를 통해 다른 기능을 실현하고 있기 때문이다.
실제 함수 오버로딩을 구현하는 방법으로는 arguments, prototype 객체 등을 활용하는 방법이 있다.
